# 常见错误

> 更新时间：2024-02-13

## 安装错误

### "App is damaged and can't be opened"（macOS）

**原因**：macOS Gatekeeper 隔离属性

**解决**：

```bash
sudo xattr -rd com.apple.quarantine "/Applications/霖君.app"
```

### "应用已损坏，无法打开"（macOS）

**原因**：macOS Gatekeeper 隔离属性

**解决**：

```bash
xattr -cr /Applications/霖君.app
```

### "Windows Defender SmartScreen 已阻止运行"

**原因**：应用未签名或签名不被信任

**解决**：点击"更多信息" > "仍要运行"

## 代理错误

### "端口已被占用"

**原因**：端口 8310（默认）被其他程序使用

**解决**：

1. 更换代理端口，或
2. 关闭占用端口的程序：

```bash
# 查找占用端口的进程（以 8310 为例）
# macOS/Linux
lsof -i :8310

# Windows
netstat -ano | findstr :8310
```

### "代理二进制未找到"

**原因**：CLIProxyAPIPlus 二进制未下载或被删除

**解决**：在 **About** 页面重新下载代理二进制

### "代理启动超时"

**原因**：系统资源不足或杀毒软件拦截

**解决**：

1. 检查系统资源使用情况
2. 将霖君添加到杀毒软件白名单
3. 尝试重启应用

## 网络错误

### "无法连接到提供商"

**原因**：网络问题或提供商服务中断

**解决**：

1. 检查网络连接
2. 检查提供商服务状态
3. 尝试使用代理/VPN

### "请求超时"

**原因**：网络延迟高或服务响应慢

**解决**：

1. 检查网络质量
2. 稍后重试
3. 考虑使用更近的服务节点

## 认证错误

### "Token 已过期"

**原因**：OAuth Token 过期

**解决**：在 Providers 页面重新进行 OAuth 登录授权

### "API 密钥无效"

**原因**：自定义提供商的 API Key 错误或已失效

**解决**：检查并更新 API Key

## CLI 工具错误

### 配置后 CLI 工具报错

**原因**：代理未运行或配置错误

**解决**：

1. 确认霖君代理服务正在运行（状态显示"运行中"）
2. 确认至少有一个提供商账号已添加
3. 检查端口 8310 是否正确

### 自动配置未检测到工具

**原因**：CLI 工具未正确安装

**解决**：

1. 确认 CLI 工具已正确安装
2. 在终端中运行工具确认可用
3. 使用手动配置作为备选方案

## 数据错误

### "配置文件损坏"

**原因**：配置文件格式错误

**解决**：

1. 备份当前配置文件
2. 删除配置文件
3. 重启应用生成默认配置
4. 重新进行必要配置

### Token 文件损坏

**原因**：`~/.cli-proxy-api/` 中的 JSON 文件损坏

**解决**：

1. 删除对应的 Token 文件
2. 重新进行 OAuth 登录

## 获取帮助

如果以上方案无法解决问题：

1. 收集错误日志
2. 在 [GitHub Issues](https://github.com/wangdabaoqq/LinJun/issues) 提交问题
3. 附上系统信息和错误日志
