# 日志

> 适用版本：v1.0.0+
> 更新时间：2024-02-13

## 你将完成

- 查看和筛选请求日志
- 定位失败请求的原因

## 前置条件

- 代理服务正在运行
- 有请求记录

## 概述

Logs 页面提供完整的 API 请求历史记录，支持大数据量下的高性能浏览（虚拟化表格和增量轮询）。

## 操作步骤

### 查看日志

1. 点击左侧菜单 **Logs**
2. 日志按时间倒序显示
3. 表格支持虚拟滚动，大量日志也能流畅浏览

### 筛选日志

支持的筛选条件：

- **时间范围**：选择开始和结束时间
- **提供商**：筛选特定提供商的请求
- **状态**：成功/失败
- **关键词**：搜索请求内容

### 查看详情

点击任意日志行，展开查看：

- 完整请求头
- 请求体
- 响应状态
- 错误信息（如果有）
- Token 使用量

## 日志字段说明

| 字段 | 含义 |
|------|------|
| Time | 请求发起时间 |
| Provider | 请求的目标提供商 |
| Account | 使用的账号 |
| Model | 使用的模型 |
| Status | HTTP 状态码 |
| Latency | 请求总耗时 |
| Tokens | Token 使用量 |

## 性能优化

霖君针对大量日志做了性能优化：

- **虚拟化表格**：只渲染可见区域，支持数万条日志流畅滚动
- **增量轮询**：只获取新增日志，减少数据传输

## 验证结果

- **预期看到**：日志列表显示请求记录
- **若未出现**：确认代理正在运行且有请求经过

## 常见问题

### 日志为空

- 确认代理服务已启动
- 确认 CLI 工具正确配置了代理地址（`http://localhost:8310`）
- 发送一个测试请求

### 找不到特定请求

- 扩大时间范围
- 使用关键词搜索
- 检查提供商筛选条件

### 日志加载慢

- 缩小时间范围以减少数据量
- 使用筛选条件定位特定日志
